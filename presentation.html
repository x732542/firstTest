Motivation
為什麼要做這個？
 現有資源的痛點
看文件讀到懂，但手邊沒有地方快速試。MDN 太詳盡，教學影片太慢，想要一個「看到就能改、改了就能看」的環境。
 核心想法
把常用的 HTML、CSS、JavaScript 語法整理成可以互動的小範例，每次練習都有即時視覺回饋，降低「看懂但不會用」的落差。
 目標使用者
剛入門前端、想要一個不必搭環境就能練習的人——打開瀏覽器就是沙盒，隨時可以改程式碼。
 AI 協作實驗
同步探索人 × AI 協作開發的可能性——80% 程式碼透過 Gemini／Claude 輔助生成，親身驗證 AI 輔助開發的效率邊界。
Challenges
製作途中的難題
🔒
沙盒安全限制
使用者輸入的程式碼直接執行，需要隔離 DOM 副作用、清除計時器與事件監聽，避免多次執行互相干擾。
🔄
狀態清除問題
切換課程後上一題的 setTimeout / setInterval 仍在背景跑，需追蹤所有 ID 並在 clearRuntime() 中統一清除。
📐
版面一致性
三個模組（HTML / CSS / JS）各自獨立 HTML 檔，要維持統一的視覺語言和交互體驗需要反覆對齊。
🤖
AI 生成程式碼的審核
AI 給的程式碼常有命名不一致或邏輯漏洞，每段都需要自己讀懂並修正，否則執行沙盒會直接報錯。
⚡
26 課程的擴充設計
從最初 5 個範例擴充到 26 個課程，要讓資料結構可擴展、UI 不改動的情況下直接新增，需要統一的 jsLessons 物件格式。
🎨
CSS 即時預覽同步
CSS 實驗室需要讓滑桿、色票、下拉選單即時反映到預覽區，事件監聽的掛載時機和 DOM 更新順序需要精確控制。

Module 01 · HTML
HTML 常用標籤教學工具

以分類索引整理最常用的 HTML 標籤，每個標籤包含語意說明、類比、解剖、樣式範例與 JS 連結，讓使用者直接修改範例程式碼並即時預覽結果。

標籤分類
文件基礎
html, head, body, title, meta, link, script
版面語意
header, nav, main, section, article, aside, footer, div
文字排版
h1~h6, p, span, a, strong, em, blockquote
表單互動
form, input, button, select, textarea, label
每個標籤包含的內容
語意 標籤的正確使用場景與語意說明
類比 用生活比喻幫助理解抽象概念
解剖 語法組成與各部分作用說明
即時預覽 直接修改範例程式並觀察渲染結果

blur / spread / RGBA / 多層陰影

Module 03 · JavaScript
JS 邏輯實驗室

延續 HTML 教學頁的版面規格，保留 JavaScript 頁面的暖橘互動感與編輯器氛圍。左側挑主題，中間看觀念，右側直接執行範例與觀察狀態變化。

實驗室功能
可直接編輯的程式碼區 Ctrl+Enter 快捷鍵執行
互動預覽區 按鈕、輸入框、列表、進度條等元件
內建 Console log / warn / error 沙盒輸出
環境重置 一鍵還原預覽區所有狀態
沙盒執行機制
function runJS() { // 用 new Function 建立隔離沙盒 const execute = new Function( 'console', 'helpers', code ); execute(sandboxConsole, createHelpers()); // 自動清理 timeout / 事件 clearRuntime(); }
每次執行前自動清除前一次的 setTimeout、setInterval 與 addEventListener，確保範例不互相干擾。
